| 接口路径                   | 方法 | 功能           | 状态 |
| -------------------------- | ---- | -------------- | ---- |
| `/api/v1/auth/login/wx`    | POST | 微信小程序登录 | ✅   |

- Refresh Token Redis 存储与验证
- 用户信息脱敏

### 3.5 检测记录拉取 (3.3.5) ✅

**Controller**: `RecordController.java`


| 接口路径          | 方法 | 功能             | 状态 |
| ----------------- | ---- | ---------------- | ---- |
| `/api/v1/records` | GET  | 分页查询检测记录 | ✅   |

**核心功能**:

- 分页查询检测记录
- 多条件筛选（时间范围、场景类型、设备 SN、等级）
- 按检测时间倒序排列
- 仅查询当前用户的数据

**已创建文件**:

- `RecordQuery.java` - 查询参数 DTO
- `RecordListVO.java` - 列表响应 VO
- `IRecordService.java` - 添加查询方法
- `RecordServiceImpl.java` - 实现查询逻辑

**查询参数**:

- `pageNum` - 页码（默认 1）
- `pageSize` - 每页数量（默认 10）
- `startTime` - 开始时间
- `endTime` - 结束时间
- `sceneType` - 场景类型（0 未选 1 空腹 2 餐后 3 运动后）
- `deviceSn` - 设备 SN
- `level` - 等级（1-5）

## 五、数据库表

### 5.1 已创建的表


| 表名              | 中文名         | 用途               | 状态 |
| ----------------- | -------------- | ------------------ | ---- |
| `sys_user`        | 用户表         | 存储用户基本信息   | ✅   |
| `t_device`        | 设备信息表     | 存储设备基础信息   | ✅   |
| `t_user_device`   | 设备绑定关系表 | 用户与设备绑定关系 | ✅   |
| `t_record`        | 检测记录表     | 存储检测数据       | ✅   |
| `t_feedback`      | 用户反馈表     | 用户反馈内容       | ✅   |
| `t_system_config` | 系统配置表     | 动态配置参数       | ✅   |

### 5.2 核心表结构

**t_record (检测记录表)**:

- `record_id` - 客户端生成的唯一记录 ID(UUID)
- `user_id` - 用户 ID
- `device_sn` - 设备 SN
- `timestamp` - 检测时间
- `scene_type` - 场景标签（0 未选 1 空腹 2 餐后 3 运动后）
- `concentration` - 浓度值 (ppb)
- `level` - 等级（1-5）
- `level_label` - 等级文案
- `suggestion` - 建议文案
- `r_base`, `r_gas` - 基线电阻、响应电阻
- `adc_value` - ADC 采样值
- `temperature`, `humidity` - 环境温度/湿度
- `heater_temp` - 加热片温度
- `algo_version`, `firmware_version` - 算法版本、固件版本
